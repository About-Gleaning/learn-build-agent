创建一个全新的本地文本文件。

## 使用说明
- 该工具仅用于创建新文件，绝不覆盖已有文件。
- 如果目标文件已经存在，工具会拒绝执行，并明确提示你改用 edit_file。
- filePath 必须是绝对路径；工具会自动创建缺失的父目录。
- 适用场景：首次创建源码文件、配置文件、文档骨架或测试文件。
- 始终优先编辑已有文件，除非明确需要创建新文件。
- 不要主动创建文档文件（如 *.md）或 README 文件，只有在用户明确要求时才创建。
- 除非用户明确要求，否则不要在文件中写入 emoji。

## 大文件策略
- 单次 content 应只写最小可运行骨架（类/函数签名，函数体留 pass 或空实现）。
- 骨架写完后，立即通过多次调用 edit_file 逐段填充实现细节，而非一次性传入完整内容。
- 这样做可以避免序列化超长内容时的错误，也便于逐步验证每段代码的正确性。

## 最佳实践示例
新建 src/server.py 时：

  # 正确：先写骨架
  content = """
  import os

  class Server:
      def start(self):
          pass

      def stop(self):
          pass
  """

  # 再用 edit_file 逐步填充 start()、stop() 的具体实现
