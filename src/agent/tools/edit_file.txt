执行文件中的精确字符串替换。

## 使用说明

- 在进行编辑之前，你必须至少使用一次 `read_file` 工具读取文件。否则该工具会报错。
- 在根据 read_file 工具输出进行编辑时，必须严格保留缩进（空格 / tab），并且以“行号前缀之后”的内容为准。
  - 行号前缀格式为：空格 + 行号 + tab
  - tab 之后的内容才是真正的文件内容
  - `oldString` 和 `newString` 中绝不能包含任何行号前缀内容
- 始终优先编辑已有文件，除非明确要求，否则不要创建新文件
- 除非用户明确要求，否则不要在文件中添加 emoji
- 如果 `oldString` 在文件中找不到，编辑会失败，并报错："oldString not found in content"
- 如果 `oldString` 在文件中出现多次，编辑会失败，并报错："oldString found multiple times and requires more code context to uniquely identify the intended match"
  - 解决方法：
    - 提供更长的上下文使其唯一
    - 或使用 `replaceAll` 替换所有匹配项
- 当需要在整个文件中替换或重命名字符串（例如变量重命名）时，应使用 `replaceAll`
