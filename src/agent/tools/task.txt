启动一个新的 agent，让它自主处理复杂的多步骤任务。

可用的 agent 类型，以及它们可访问的工具如下：  
`{agents}`

## 何时使用 Task 工具

- 当你被要求执行自定义 slash 命令时。  
  这时应使用 Task 工具，并将 slash 命令调用本身作为完整 prompt。slash 命令可以带参数。  
  例如：

```text
Task(prompt="/check-file path/to/file.py")
````

## 何时不要使用 Task 工具

* 如果你想读取某个具体文件路径，应使用 Read 或 Glob 工具，而不是 Task 工具，这样能更快找到目标
* 如果你在查找某个具体类定义，比如 `class Foo`，应使用 Glob 工具，这样更快
* 如果你要在某个特定文件或 2-3 个文件中搜索代码，应使用 Read 工具，而不是 Task 工具，这样更快
* 其他不属于上述 agent 描述范围的任务

---

## 使用说明

1. 只要可能，就并发启动多个 agent，以最大化性能；要做到这一点，应在同一条消息中包含多个工具调用

2. agent 完成后，会只返回一条消息给你。
   agent 返回的结果对用户不可见。
   如果你想让用户看到结果，你应再发一条文本消息给用户，对结果做简洁总结。

3. 你的 prompt 应包含一段**非常详细的任务说明**，让 agent 能够自主执行；并且你应明确指定，希望它在最终且唯一的一条返回消息中向你返回哪些信息。

4. 一般来说，应当信任 agent 的输出结果

5. 要明确告诉 agent，你希望它是**编写代码**，还是仅做**研究工作**（如搜索、读文件、抓取网页等），因为它并不知道用户的真实意图

6. 如果某个 agent 的描述里提到应当主动使用，那么你应尽量主动调用它，而不必等用户先提出要求。请自行判断。

---

## 使用示例

> 注意：下面的 agent 只是虚构示例，用于说明；实际使用时应以你上方列出的真实 agent 为准。

### 示例 agent 描述

```text
"code-reviewer": 在你完成一段较重要的代码编写后使用这个 agent
"greeting-responder": 当用户打招呼时，使用这个 agent 用友好的玩笑来回应
```

### 示例 1

用户：

```text
请写一个函数，用来检查一个数字是否为质数
```

助手：

```text
好的，我来写一个检查数字是否为质数的函数
```

助手：

```text
首先我会使用 Write 工具来写一个检查数字是否为质数的函数
```

助手：

```text
我将使用 Write 工具写入以下代码：
```

```js
function isPrime(n) {
  if (n <= 1) return false
  for (let i = 2; i * i <= n; i++) {
    if (n % i === 0) return false
  }
  return true
}
```

说明：

```text
由于已经写完一段较重要的代码，并且任务已经完成，现在应使用 code-reviewer agent 来审查代码
```

助手：

```text
现在我将使用 code-reviewer agent 来审查代码
```

助手：

```text
使用 Task 工具启动 code-reviewer agent
```

### 示例 2

用户：

```text
Hello
```

说明：

```text
由于用户是在打招呼，因此应使用 greeting-responder agent，用友好的玩笑进行回应
```

助手：

```text
我将使用 Task 工具来启动 greeting-responder agent
```
