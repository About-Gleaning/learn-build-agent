## 必须保留的信息

- 接口名称、Method、Path
- 鉴权方式和通用请求约束
- 参数位置、类型、是否必填、默认值、枚举/格式限制
- 请求示例、响应字段、响应示例
- 错误码或失败语义
- 与实现直接相关的业务规则

## 可以保留的信息

- 文档版本
- 变更记录
- 接口调用时序
- 环境差异说明

## 不要保留的信息

- 页眉页脚
- 重复目录
- 宣传性介绍
- 无意义截图转写
- 空表格
- 明显 OCR 脏数据

## 写作要求

- 统一使用中文说明
- 用表格表达参数和错误码，便于后续程序开发
- 用代码块保留请求/响应样例
- 对不确定内容标记 `待确认`，不要编造
